write_type(obj, where=None)[source]¶

Write the type of the object being saved to the hdf5 file. This allows for automatic loading of a file into a specific DOLfYN class.

When a file is loaded, a dict of string:class (key:val) pairs should be provided to the ‘loader’ data_factory. The string (key) that matches the value of the file’s ‘_object_type’ attribute is chosen. The corresponding class (value) is then instanced and data is loaded into it according to the DOLfYN specification.

This function writes the ‘_object_type’ from the current DOLfYN instance to the file so that it can be loaded later.

See also: - loader.read_type - get_typemap
